Intentionally vulnerable endpoints for security training. Every action is monitored in real-time by the admin dashboard.
// Active vulnerabilities:
SQL Injection (login, search) · Stored XSS (posts) · Reflected XSS (/api/greet) · IDOR (user profiles) · Path Traversal (/api/file) · Command Injection (/api/ping) · Broken Access Control (/api/secrets)
💉 SQL Injection — Search Critical
The search endpoint concatenates raw strings. Use UNION SELECT to extract data from other tables including secrets and flags.
📝 Stored XSS — Posts Critical
Posts are stored and rendered as raw HTML with no sanitization. Inject a script that fires every time the posts feed loads.
📂 Path Traversal Critical
The /file endpoint joins paths without sanitization. Escape the web root using ../ sequences to read server files.
💻 Command Injection — Ping Critical
User input is passed directly to exec(). Chain system commands using semicolons, pipes, or backticks.
🔓 IDOR — User Enumeration High
User profiles are accessible by numeric ID with no ownership check. Enumerate all users by incrementing the ID.
🗝️ Broken Access Control Critical
The /secrets endpoint requires no authentication whatsoever. Access all server secrets, API keys, and CTF flags directly.
